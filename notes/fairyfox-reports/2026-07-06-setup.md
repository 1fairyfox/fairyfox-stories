## What was done

Bootstrapped Fairy Fox Stories from scratch as an AI **story farm**. Cleared a prior
unauthorized scaffold (greenfield), pulled the hub read-only, adopted the standards + notes
skeleton, and built:

- The **craft body of knowledge** (`notes/reference/craft/`) — researched, sourced.
- The farm's **operating model**, **universe system**, and **reading-experience** specs.
- A **Jekyll** site (deliberate deviation from the games repo's static HTML — see below):
  collections for books, chapters, universes, characters, cities; layouts + the shared mesh

  hand-drafted chapter 1, a full public blueprint, and its universe/character/city entries.

Verified: `npm test` green (5 checks); `bundle exec jekyll build` clean; all pages generate and
cross-link. Committed on `dev`; **stopped before `main`** for Fairy Fox's release call.

## What was rough / didn't fit a fresh repo

- **Runbook assumes static-or-one-app.** This project is both a **collection** *and* a **Jekyll**
  site with several **content collections** (books/chapters/universes/characters/cities). The
  runbook's step 3 template copy and step 6 docs-site guidance are written for a single static
  app; a collection-of-content-types Jekyll node needed real adaptation the runbook doesn't cover.
- **"Self-contained + liftable unit" vs. meshing.** The mesh's usual per-unit self-containment
  rule (games) fights the owner's goal of a cross-linked world. Recorded as a deliberate decision
  (`decisions/architecture.md`); the runbook could acknowledge "relational content site" as a node
  shape.
- **`node --test <dir>`** isn't valid in Node 24 (MODULE_NOT_FOUND); needed a glob
  (`node --test "scripts/**/*.test.mjs"`). Worth noting in agent-tooling.
- **Legal templates** are written around "games/best scores"; re-scoping to "stories/reading
  progress" was manual. A more neutral legal template would help any non-games node.

## Suggestions for the system

1. Add a **"content-collection Jekyll node"** variant to `new-project-setup` (collections,
   layouts, the meshing pattern) — both story/knowledge nodes will want it.
2. Note the **Node 24 `--test` glob** requirement in `agent-tooling.md`.
3. Provide a **medium-neutral legal template** (data practices only), so nodes don't hand-edit
   games-specific wording.
